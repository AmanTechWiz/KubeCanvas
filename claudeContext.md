KubeCanvas is a real-time collaborative system design workspace where users describe a system in plain English, and an AI agent (KubeAI) maps it onto a shared canvas built with React Flow and Liveblocks — collaborators can then refine the architecture together and generate a technical spec from the graph. The AI pipeline uses Trigger.dev v4 to run a durable background task that calls Google Gemini via the Vercel AI SDK with structured tool calls (addNode, addEdge, moveNode, etc.) to incrementally build the diagram, while a backend-driven AI cursor animates on the canvas via Liveblocks Storage and REST API so users watch the architecture take shape in real time. The key question: what's the best approach to get clean, professional diagrams with proper grid alignment, connected nodes, and readable edges — should the AI generate raw x/y positions, or should a post-generation layout algorithm (like dagre/elkjs) handle positioning, or should the tool design shift to a simpler layer/column coordinate system?
